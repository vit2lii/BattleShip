# Contributing

Дякую за інтерес до **BattleShip**.

## Як запропонувати зміни
1. Створіть гілку від `main`.
2. Зробіть зміни маленькими та зрозумілими (одна логічна зміна на коміт).
3. Додайте/оновіть тести, якщо зміна впливає на логіку.
4. Переконайтесь, що проєкт збирається і тести проходять.

## Збірка і тести (приклад)
```bash
cmake -S . -B build -G Ninja -DBattleShip_BUILD_EXECUTABLE=ON -DBattleShip_ENABLE_UNIT_TESTING=ON
cmake --build build
ctest --test-dir build --output-on-failure
```

## Стиль коду
- Дотримуйтесь існуючого стилю.
- Перед PR можна прогнати форматування (якщо ціль `clang-format` доступна у вашій конфігурації збірки).

## Повідомлення про проблеми
Коли створюєте Issue, додайте:
- короткий опис проблеми
- кроки для відтворення
- очікуваний/фактичний результат
- вашу ОС, компілятор, версію CMake
